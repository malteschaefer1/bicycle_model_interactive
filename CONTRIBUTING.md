# Contributing / Beitragen

Thanks for your interest in improving this project.

## English

### Before You Start

- Keep changes simple and easy to explain.
- Avoid large unrelated refactors.
- Open an issue first for bigger changes.

### Setup

1. Fork and clone the repository.
2. Run the quality gate:

```bash
./scripts/run_checks.sh
```

3. Open `model/bicycle_model.html` in a browser for manual checks.

### Development Rules

- Keep plain HTML/CSS/vanilla JavaScript style.
- Prefer small, reviewable commits.
- Add or update tests for behavior changes.
- Update relevant docs when behavior changes.

### Pull Request Checklist

- [ ] Tests pass locally (`./scripts/run_checks.sh`)
- [ ] Scope is focused
- [ ] Docs updated
- [ ] No secrets or credentials added
- [ ] Manual browser check completed
